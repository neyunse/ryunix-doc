En apps de producción, omite `@canary` o usa `--latest` en Create Ryunix App para resolver las etiquetas estables de npm.

## Paquetes publicados en npm

RyunixJS se divide en paquetes con responsabilidades claras. CRA resuelve las etiquetas `latest` o `canary` del registro al generar el proyecto.

<NpmPackagesTable locale="es" />

| Instalación | Comando |
| :--- | :--- |
| Scaffolding (canary) | `npx @unsetsoft/cra@canary my-app --canary` |
| Scaffolding (estable) | `npx @unsetsoft/cra my-app` o `npx @unsetsoft/cra my-app --latest` |
| Solo core + presets | Ver pestañas de gestores de paquetes arriba |
