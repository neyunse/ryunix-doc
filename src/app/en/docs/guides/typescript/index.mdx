# TypeScript

RyunixJS ships TypeScript definitions on npm for `@unsetsoft/ryunixjs` and `@unsetsoft/ryunix-presets`. CRA templates still scaffold JavaScript apps (`.ryx`), but you can add TypeScript or get editor autocomplete with JSDoc without migrating the whole project.

## Available types

| Need | Package / file |
| :--- | :--- |
| Typed `ryunix.config.js` | `@unsetsoft/ryunix-presets` ã `RyunixUserConfig` |
| Hooks, components, SSR | `@unsetsoft/ryunixjs` ã `types/index.d.ts` |
| Automatic JSX | `"jsxImportSource": "@unsetsoft/ryunixjs"` in `tsconfig.json` |

## Typed config with JSDoc

The quickest path is typing `ryunix.config.js` with a JSDoc comment (CRA templates already include this):

```javascript
/** @type {import('@unsetsoft/ryunix-presets').RyunixUserConfig} */
export default {
  ssr: true,
  port: 3000,
};
```

Your editor resolves `RyunixUserConfig` from the `.d.ts` files published in `@unsetsoft/ryunix-presets`.

## Full TypeScript project

1. Install TypeScript as a dev dependency:

```bash
pnpm add -D typescript
```

2. Create `tsconfig.json` at your app root:

```json
{
  "compilerOptions": {
    "target": "ES2022",
    "module": "ESNext",
    "moduleResolution": "bundler",
    "jsx": "react-jsx",
    "jsxImportSource": "@unsetsoft/ryunixjs",
    "allowJs": true,
    "checkJs": false,
    "strict": true,
    "noEmit": true,
    "skipLibCheck": true,
    "paths": {
      "@/*": ["./src/*"]
    }
  },
  "include": ["app/**/*.ryx", "src/**/*", "ryunix.config.js"]
}
```

3. Import types from the core when writing components:

```typescript
import {
  useStore,
  createElement,
  type RyunixElement,
} from "@unsetsoft/ryunixjs";
```

> [!WARNING]
> Route files remain `.ryx` (JavaScript + JSX). Ryunix compiles with Babel/SWC; TypeScript is for static checking and editor support, not the framework's primary compiler.

## `jsconfig.json` in CRA templates

Official templates include `jsconfig.json` with `@/` aliases and type resolution. That is enough for VS Code autocomplete without adding TypeScript explicitly.

Pair it with the [VS Code extension](/en/docs/guides/vscode) for go-to-definition, hover, and diagnostics in `.ryx` files.
