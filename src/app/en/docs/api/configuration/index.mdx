# Configuration

Configure your RyunixJS project with `ryunix.config.js`.

## Basic Configuration

```javascript
// ryunix.config.js
/** @type {import('@unsetsoft/ryunix-presets').RyunixUserConfig} */
const RyunixSettings = {
  rootDir: "src", // omit if app/ is at project root; default is "src"
  ssr: true,
  mdx: true,
  style: true,
  port: 3000,
  buildDir: ".ryunix",
  favicon: true,
  webpack: {
    production: false,
    resolve: {
      alias: {
        "@": "./src",
        "@/components": "./src/components",
      },
    },
  },
};

export default RyunixSettings;
```

> [!WARNING]
> **Legacy keys:** `experimental.ssr`, `experimental.mdx`, `static.favicon`, and `webpack.output.buildDirectory` still work but log deprecation warnings. Prefer the root-level options above (`ssr`, `mdx`, `favicon`, `buildDir`, `port`, `proxy`).

## Webpack Settings

### Production Mode

```javascript
{
  webpack: {
    production: true, // Enable production optimizations
  },
};
```

Production mode enables:

- Minification (Terser + CSS)
- Code splitting
- Tree shaking
- Source maps

### Source root (`rootDir`)

Ryunix looks for `app/` at the project root **or** at `{rootDir}/app`. Default `rootDir` is `"src"`.

```javascript
{
  rootDir: "src", // routes live in src/app/
}
```

If both `./app` and `./src/app` exist, **`./app` wins**.

### Build output (`buildDir`)

```javascript
{
  buildDir: ".ryunix",
}
```

Production static files are written to `.ryunix/static/`.

### Dev server (`port`, `proxy`)

```javascript
{
  port: 3000,
  proxy: {
    "/api": {
      target: "http://localhost:8080",
      changeOrigin: true,
    },
  },
  webpack: {
    devServer: {
      allowedHosts: "auto",
    },
  },
};
```

### Path Aliases

```javascript
{
  webpack: {
    resolve: {
      alias: {
        "@": "./src",
        "@/components": "./src/components",
      },
    },
  },
};
```

Use in code:

```jsx
import Button from "@/components/Button";
import { formatDate } from "@/utils/date";
```

### Extensions

```javascript
{
  webpack: {
    resolve: {
      extensions: [".js", ".jsx", ".ryx", ".ts", ".tsx"],
    },
  },
};
```

### Custom Webpack Rules

```javascript
{
  webpack: {
    module: {
      rules: [
        {
          test: /\.svg$/,
          use: ["@svgr/webpack"],
        },
        {
          test: /\.yaml$/,
          use: ["yaml-loader"],
        },
      ],
    },
  },
};
```

### Plugins

```javascript
const CopyPlugin = require("copy-webpack-plugin");

{
  webpack: {
    plugins: [
      new CopyPlugin({
        patterns: [{ from: "static", to: "static" }],
      }),
    ],
  },
};
```

## Static Assets

### Favicon

```javascript
{
  favicon: true, // uses public/favicon.png when present
}
```

### Custom Template (Deprecated)

> [!WARNING]
> **Deprecated Feature:** Modifying root attributes via `customTemplate` is obsolete in the App Router.
>
> **Replacement:** Use a base `app/layout.ryx` (root layout) to define the main skeleton connecting to the DOM.

```javascript
{
  static: {
    favicon: true,
    customTemplate: true, // Deprecated
  },
};
```

### SEO Configuration (Deprecated)

> [!WARNING]
> **Deprecated Feature:** Global SEO configuration in `ryunix.config.js` via `static.seo` is obsolete.
>
> **Replacement:** Export a local `Metatags` object in your base components, `layout.ryx`, or use YAML Frontmatter in `.mdx` files.

```javascript
{
  static: {
    seo: { // Deprecated
      pageLang: "en",
      title: "My App",
    },
  },
};
```

## MDX & SSG

### MDX support

Enable `.mdx` pages under `app/`:

```javascript
{
  mdx: true,
}
```

YAML headers (like `title`, `description`) placed at the start of `.mdx` files between `---` are parsed and invisibly transpiled. Ryunix assimilates them directly as the `Metatags` constant, injecting them into the HTML and optimizing SEO without requiring additional wrappers.

### Global styles (`style`)

Defaults to `style: true`. Ryunix injects `@unsetsoft/ryunixjs/style.css` for semantic HTML across the whole app (`.ryx`, `.mdx`, and native tags).

```javascript
{
  style: true, // default; set false to disable
}
```

**Light / dark theme:** toggle only via the root element class. No `dark` on `<html>` → light; `class="dark"` → dark.

```javascript
document.documentElement.classList.toggle("dark");
```

**Opt-out** without disabling global styles:

- `data-ryx-unstyled` on any element or wrapper.
- `unstyled` prop on components (maps to `data-ryx-unstyled` via core `ryxProps`).

`ryunix-tailwind` and `ryunix-all` templates ship with `style: false` to avoid conflicting with Tailwind.

### Static site generation (SSG)

With the App Router, `ryunix build` writes a route manifest to `.ryunix/cache/ssg/routes.json` and prerenders static HTML for routes **without** dynamic segments (`[param]` paths are SSR-only unless listed manually).

Optional sitemap when no `app/sitemap.js` exists:

```javascript
{
  legacy: {
    ssg: {
      sitemap: {
        enable: true,
        baseURL: "https://example.com",
        settings: {
          changefreq: "weekly",
          priority: "0.7",
        },
      },
    },
  },
}
```

Prefer `app/sitemap.js`, `app/robots.js`, and `app/manifest.js` when using file-based metadata (see [Special files](/en/docs/api/special-files)).

## Deprecations & Bugfixes

- **Centralized Routing (`routes.ryx`)**: Webpack previously threw a critical `[SSG] ❌ The route file was not found` error if `routes.ryx` was missing. Now, if the `app/` directory is detected, this legacy check is silenced and skipped globally.
- **API Nested File Watcher (Hot Reload Bugfix)**: Fixed an issue where compiling API endpoints inside subfolders failed due to an overly aggressive system clock (mtime) check. Now, modifying endpoints like `app/api/users/[id]/route.js` updates correctly on disk instead of returning `NotFound`.
- **MDX HTML Injection Bugfix**: Corrected an async issue in the SSG compiler where markdown `.mdx` components rendered a sterile default `<title>Ryunix App</title>` tag, skipping the merge step with the layout's route formats.

## Environment Variables

Variables must reflect the `RYUNIX_APP_` prefix to be available in the client.

### From `ryunix.config.js`

```javascript
const RyunixSettings = {
  env: {
     RYUNIX_APP_TEST: "example",
  },
};

export default RyunixSettings;
```

Access in code:

```javascript
const key = ryunix.config.env.RYUNIX_APP_KEY;
```

### From `.env` File

```env
RYUNIX_APP_TEST=test
```

Access in code:

```javascript
const myEnv = ryunix.env.RYUNIX_APP_TEST;
```
