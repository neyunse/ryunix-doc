# Create Ryunix App

Usa Create Ryunix App (`@unsetsoft/cra`) para generar un nuevo proyecto a partir de plantillas oficiales.

## Inicio rápido

<CraQuickStart channel="canary" />

Equivalente con otros gestores de paquetes:

```bash
pnpm create @unsetsoft/cra my-app
yarn create @unsetsoft/cra my-app
bun create @unsetsoft/cra my-app
```

## Plantillas oficiales

| Plantilla | Incluye |
| :--- | :--- |
| `ryunix-base` | App mínima con App Router |
| `ryunix-tailwind` | PostCSS + Tailwind (`@tailwindcss/postcss`) |
| `ryunix-eslint` | ESLint alineado con convenciones Ryunix |
| `ryunix-all` | Tailwind + ESLint combinados |

El CLI copia la carpeta de plantilla elegida tal cual en tu directorio destino (incluyendo `app/`, `ryunix.config.js` y `public/`).

## Flags de línea de comandos

Para omitir los prompts interactivos, puedes pasar flags directamente:

| Flag | Descripción |
| :--- | :--- |
| `--canary` | Usa el canal Canary para `@unsetsoft/ryunixjs` y `@unsetsoft/ryunix-presets` |
| `--latest` | Usa el canal Latest (estable; predeterminado) |
| `--tailwind` | Plantilla con PostCSS + Tailwind (`@tailwindcss/postcss`) |
| `--eslint` | Plantilla con ESLint alineado a convenciones Ryunix |
| `--vscode` | Escribe `.vscode/` con extensiones recomendadas (Ryunix + ESLint) |
| `--compiler <swc\|babel>` | Compilador en `ryunix.config.js` (predeterminado: `swc`) |

Ejemplos:

```bash
# App completa: Tailwind + ESLint + VS Code, canal canary
npx @unsetsoft/cra@canary my-app --canary --tailwind --eslint --vscode

# Solo Tailwind, compilador Babel
npx @unsetsoft/cra@canary my-app --tailwind --compiler babel
```

Con `--vscode` y `--eslint`, CRA también recomienda Prettier y configura Emmet, asociación `*.ryx` y el probe ESLint para el lenguaje `ryunix`. Consulta [VS Code](/es/docs/guides/vscode) para detalles de la extensión.

## Estructura generada

Salida típica:

```txt
my-app/
├── app/
│   ├── index.ryx
│   ├── layout.ryx
│   ├── error.ryx      # 404 global (export NotFound)
│   └── api/hello/router.js
├── public/
├── styles/
├── ryunix.config.js
└── package.json
```

## Después del scaffolding

1. Ejecuta `pnpm dev` y abre `http://localhost:3000`.
2. Ajusta alias en `ryunix.config.js` (`@/styles`, `@/components`, …).
3. Añade rutas bajo `app/` — las carpetas se mapean a URLs automáticamente.

## Canary vs estable

La documentación y ejemplos suelen usar `@unsetsoft/cra@canary` para las últimas funciones del framework. Fija una versión estable para apps de producción cuando haya releases etiquetados.

<CanaryChannelVersions />

Con `--canary`, CRA ejecuta `npm view @unsetsoft/ryunixjs@canary version` (y lo mismo para presets) y escribe rangos semver en `package.json`. Con `--latest` (predeterminado), resuelve la etiqueta `latest` del registro.
