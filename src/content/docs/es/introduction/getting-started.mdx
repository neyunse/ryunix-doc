# Primeros pasos

RyunixJS es un framework JavaScript ligero para construir interfaces de usuario. Como React, pero más pequeño y con routing integrado.

Este sitio de documentación usa el mismo App Router, MDX y metadatos que verás en [SEO y metadatos](/es/docs/guides/seo-metadata):

<img src="/screenshot.png" alt="Página de inicio de la documentación de RyunixJS" width="1830" height="1076" loading="lazy" />

> [!WARNING]
> **Routing integrado** (la forma heredada de definir rutas manualmente) ya no está soportado. Usa el **App Router** en su lugar.

## Inicio rápido

Tu aplicación ya está en ejecución en `http://localhost:3000`

## Tu primer componente

```jsx
// app/index.ryx
import { useStore } from "@unsetsoft/ryunixjs";

function App() {
  const [count, setCount] = useStore(0);

  return (
    <div>
      <h1>Count: {count}</h1>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  );
}

export default App;
```

## Conceptos clave

- **Componentes** - Funciones que devuelven JSX
- **Hooks** - `useStore`, `useEffect`, `useMemo`, etc.
- **Routing** - App Router integrado, sin paquetes externos
- **JSX** - Misma sintaxis que React

## Estructura del proyecto

```
my-app/
├── app/               # App Router (o src/app/ con rootDir: "src")
│   ├── layout.ryx
│   ├── index.ryx
│   └── error.ryx      # 404 global (export NotFound)
├── assets/            # Estáticos (plantilla CRA)
├── styles/
└── ryunix.config.js
```

> [!TIP]
> Si `app/` no está en la raíz del proyecto, usa `rootDir: "src"` y coloca las rutas en `src/app/`.
