# Primeros pasos

RyunixJS es un framework JavaScript **independiente** para construir interfaces web. Programas con **componentes y JSX** de forma similar a React, pero el motor es propio de Ryunix. El **App Router** (rutas por carpetas), **SSR**, **SSG**, **MDX** y la **toolchain** (`ryunix dev`, `build`, `start`) vienen integrados — no necesitas instalar React ni un router aparte.

Si vienes de **Next.js**, la estructura te resultará familiar: páginas en `app/`, layouts en `layout.ryx` y APIs en `app/api/`. Si vienes de **React**, cambia `useState` por `useStore` y olvídate de configurar react-router manualmente.

## Tu primer componente

Un **componente** es una función que devuelve JSX — la mezcla de HTML y JavaScript que describe lo que se ve en pantalla. El hook **`useStore`** guarda un valor que puede cambiar (como un contador):

```jsx
// app/index.ryx
import { useStore } from "@unsetsoft/ryunixjs";

function App() {
  const [count, setCount] = useStore(0);

  return (
    <div>
      <h1>Count: {count}</h1>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  );
}

export default App;
```

## Conceptos clave

- **Componentes** — Funciones que devuelven JSX; son los bloques con los que construyes la interfaz.
- **Hooks** — Funciones como `useStore`, `useEffect` o `useMemo` para estado, efectos y optimización.
- **App Router** — Cada carpeta en `app/` es una ruta; no hace falta un paquete de routing externo.
- **JSX** — Misma sintaxis que React; los archivos de página usan extensión `.ryx`.
- **CLI `ryunix`** — `dev` para desarrollo con recarga en caliente, `build` para producción (y SSG), `start` para servir la app compilada.

## Estructura del proyecto

```
my-app/
├── app/               # App Router (o src/app/ con rootDir: "src")
│   ├── layout.ryx     # Layout compartido (header, footer…)
│   ├── index.ryx      # Página de inicio (/)
│   └── error.ryx      # 404 global (export NotFound)
├── assets/            # Estáticos (plantilla CRA)
├── styles/
└── ryunix.config.js   # Configuración del framework
```

> [!TIP]
> Si `app/` no está en la raíz del proyecto, usa `rootDir: "src"` y coloca las rutas en `src/app/`.
