# Introducción

**RyunixJS** es un framework JavaScript para crear **sitios web y aplicaciones interactivas**: desde un portfolio o una documentación hasta un panel de administración o una app con login, formularios y datos del servidor.

Si ya programaste algo de web (HTML, CSS, JavaScript), Ryunix te da una forma moderna de construir la interfaz **sin montar todo a mano**: rutas, estado, render en servidor y herramientas de desarrollo vienen integrados.

## ¿Qué es, en pocas palabras?

Ryunix es un **motor de interfaz de usuario** con una forma de programar **parecida a React** (componentes, JSX, hooks), pero **sin usar React por dentro**. Tiene su propio runtime — reconciliador, hooks y render — pensado para ser **ligero, rápido y fácil de mantener**.

En la práctica escribes archivos `.ryx` con JSX, organizas las páginas en carpetas bajo `app/` y usas comandos como `ryunix dev`, `ryunix build` y `ryunix start` para desarrollar y publicar.

## ¿Cómo funciona?

El flujo habitual es sencillo:

1. **Escribes componentes** — funciones que devuelven la interfaz (botones, listas, formularios).
2. **Defines rutas con carpetas** — cada carpeta en `app/` es una URL (`app/about/index.ryx` → `/about`).
3. **El CLI compila tu app** — Webpack genera bundles para el navegador y para el servidor.
4. **El runtime pinta la UI** — en el cliente, en el servidor (SSR) o como HTML estático (SSG), según lo que necesites.

No hace falta instalar React, react-router ni Next.js: el framework trae motor, routing y toolchain en paquetes `@unsetsoft/*`.

## Parentesco con React y Next.js

Ryunix **no es React ni Next.js**, pero si los conoces la curva de aprendizaje es baja.

### Si vienes de React

| Concepto | React | RyunixJS |
| :--- | :--- | :--- |
| Componentes y JSX | ✅ | ✅ Misma idea |
| Estado local | `useState` | `useStore` (mismo patrón) |
| Efectos, memo, refs | `useEffect`, `useMemo`, … | Hooks equivalentes |
| Eventos | Sistema sintético | Eventos nativos del DOM |
| Routing | Librería externa (p. ej. react-router) | **App Router integrado** por carpetas |

Ryunix **recuerda** la API de React, pero el motor es **propio**. No puedes instalar librerías que dependan de `react` como peer dependency sin adaptarlas.

### Si vienes de Next.js

| Concepto | Next.js | RyunixJS |
| :--- | :--- | :--- |
| Routing por archivos | `app/page.tsx`, `layout.tsx` | `app/index.ryx`, `layout.ryx` |
| Render híbrido | CSR, SSR, SSG | CSR, SSR, SSG |
| Full-stack | API routes, Server Actions | `app/api/.../router.js`, Server Actions |
| Comandos | `next dev`, `build`, `start` | `ryunix dev`, `build`, `start` |
| Motor de UI | React | **Motor propio** (`@unsetsoft/ryunixjs`) |

**Analogía rápida:** React es el motor de UI; Next.js es React + full-stack. En Ryunix, `@unsetsoft/ryunixjs` es el motor y `@unsetsoft/ryunix-presets` + Create Ryunix App aportan la capa full-stack.

## Ventajas propias de RyunixJS

### Arquitectura independiente

El core **no embebe React ni Preact**. Eso implica un runtime más pequeño, sin depender de los ciclos de release de terceros, y control directo sobre cómo evoluciona el framework.

### API familiar, convenciones propias

Hooks como `useEffect` o `useMemo` se sienten como en React; otros son específicos de Ryunix (`useStore`, `usePersistentStore`, `useSwitch`, `useDebounce`). Los archivos usan extensión `.ryx` y la config vive en `ryunix.config.js`.

### Full-stack integrado

App Router, SSR, SSG, Server Components, Server Actions, rutas API y MDX nativo forman parte del ecosistema oficial — no son plugins opcionales dispersos.

### Toolchain listo para trabajar

Create Ryunix App genera el proyecto en segundos. La extensión de VS Code resalta `.ryx`, DevTools inspecciona el árbol de componentes y el CLI unifica desarrollo, build y producción.

### Rendimiento y simplicidad

Reconciliador basado en fibers, eventos nativos del DOM y un núcleo sin dependencias externas en el runtime. La filosofía prioriza **claridad y mantenibilidad** frente a abstracciones innecesarias.

## ¿Para qué proyectos encaja?

- Sitios estáticos, landings y blogs (SSG)
- Documentación con MDX (como este sitio)
- SPAs y paneles interactivos
- Aplicaciones full-stack con APIs y lógica en servidor

## Filosofía

El desarrollo frontend moderno a menudo acumula complejidad: muchas capas, herramientas acopladas y cambios frecuentes de API. Ryunix busca una alternativa más directa:

> **Construye aplicaciones web modernas con control total y complejidad mínima.**

Con Ryunix obtienes lógica explícita, una experiencia de desarrollo predecible y una base pensada para proyectos que duren.
