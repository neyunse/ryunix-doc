# Getting Started

RyunixJS is a standalone JavaScript framework for building web UIs. You program with components and JSX much like React, but Ryunix ships its own engine. The App Router, SSR, SSG, MDX, and the toolchain are built in — you do not install React or a separate router.

If you know Next.js, the layout will feel familiar (pages under app/, layouts in layout.ryx, APIs in app/api/). If you know React, swap useState for useStore and skip manual react-router setup.

> [!WARNING]
> Integrated routing with RouterProvider is no longer supported. Use the App Router with folders under app/ instead.

## Your First Component

A component is a function that returns JSX — the mix of HTML and JavaScript that describes what appears on screen. The useStore hook holds a value that can change (like a counter):

```jsx
// app/index.ryx
import { useStore } from "@unsetsoft/ryunixjs";

function App() {
  const [count, setCount] = useStore(0);

  return (
    <div>
      <h1>Count: {count}</h1>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  );
}

export default App;
```

## Key Concepts

- Components — functions that return JSX; the building blocks of your UI.
- Hooks — functions like useStore, useEffect, or useMemo for state, effects, and optimization.
- App Router — each folder under app/ is a route; no external routing package required.
- JSX — same syntax as React; page files use the .ryx extension.
- ryunix CLI — dev for development with hot reload, build for production (and SSG), start to serve the compiled app.

## Project Structure

```
my-app/
├── app/               # App Router (or src/app/ when rootDir is "src")
│   ├── layout.ryx     # Shared layout (header, footer…)
│   ├── index.ryx      # Home (/)
│   └── error.ryx      # Global 404 (export NotFound)
├── assets/            # Static files (CRA default)
├── styles/
└── ryunix.config.js   # Framework configuration
```

> [!TIP]
> If `app/` is not at the project root, set `rootDir: "src"` in `ryunix.config.js` and place routes in `src/app/`.
