# DevTools

Ryunix provides development-time diagnostics in two layers: runtime warnings in the core bundle and an optional browser extension.

## Built-in development checks

When `NODE_ENV` is not `production`, the core includes helpers that:

- Warn on invalid hook usage (hooks outside render)
- Surface deprecation notices
- Tree-shake out of production bundles

These run automatically — no setup required.

## Ryunix DevTools extension

The **`@unsetsoft/ryunix-devtools`** package (current release **1.3.1** in the monorepo) is a Chromium browser extension that inspects Ryunix applications:

- Fiber tree and component props
- Hook state attached to fibers
- Integration with the running app via a content script

Build from `packages/ryunix-devtools` in the [Ryunixjs](https://github.com/UnSetSoft/Ryunixjs) repository and load the folder unpacked in Chrome (Manifest V3). Then open the Ryunix panel while your app runs on `localhost` (or any dev URL).

> [!NOTE]
> DevTools inspects the app in the browser. To edit `.ryx` in VS Code (syntax, LSP, snippets), use the [VS Code extension](/en/docs/guides/vscode) — they are complementary tools.

## Development overlay

On uncaught render errors in development, Ryunix may show **RyunixDevOverlay** with:

- Filtered stack traces (application code first)
- Source location when compile-time `__source` metadata exists
- Dismiss and reload to recover after a fix

Customize UX with `app/error.ryx` (404) or segment `error.ryx` files — see [Error handling](/en/docs/core/error-handling).

## Performance profiler

Use `withProfiler`, `useProfiler`, and `profiler.logStats()` in development to find slow components. Details: [Performance](/en/docs/core/performance).

## Dual-compiler errors

Ryunix runs **client** and **server** Webpack builds in parallel. Build overlays deduplicate identical errors from both compilers so you see one clear message in the browser.
