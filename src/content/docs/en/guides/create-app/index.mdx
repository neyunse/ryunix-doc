# Create Ryunix App

Use **Create Ryunix App** (`@unsetsoft/cra`) to generate a new project from official templates.

## Quick start

<CraQuickStart channel="canary" />

Equivalent with other package managers:

```bash
pnpm create @unsetsoft/cra my-app
yarn create @unsetsoft/cra my-app
bun create @unsetsoft/cra my-app
```

## Official templates

| Template | Includes |
| :--- | :--- |
| `ryunix-base` | Minimal App Router app |
| `ryunix-tailwind` | PostCSS + Tailwind (`@tailwindcss/postcss`) |
| `ryunix-eslint` | ESLint aligned with Ryunix conventions |
| `ryunix-all` | Tailwind + ESLint combined |

The CLI copies the chosen template folder verbatim into your target directory (including `app/`, `ryunix.config.js`, and `public/`).

## Command-line flags

To skip interactive prompts, pass flags directly:

| Flag | Description |
| :--- | :--- |
| `--canary` | Use the Canary channel for `@unsetsoft/ryunixjs` and `@unsetsoft/ryunix-presets` |
| `--latest` | Use the Latest (stable) channel (default) |
| `--tailwind` | Template with PostCSS + Tailwind (`@tailwindcss/postcss`) |
| `--eslint` | Template with ESLint aligned to Ryunix conventions |
| `--vscode` | Writes `.vscode/` with recommended extensions (Ryunix + ESLint) |
| `--compiler <swc\|babel>` | Compiler in `ryunix.config.js` (default: `swc`) |

Examples:

```bash
# Full stack: Tailwind + ESLint + VS Code, canary channel
npx @unsetsoft/cra@canary my-app --canary --tailwind --eslint --vscode

# Tailwind only, Babel compiler
npx @unsetsoft/cra@canary my-app --tailwind --compiler babel
```

With `--vscode` and `--eslint`, CRA also recommends Prettier and configures Emmet, `*.ryx` file association, and the ESLint probe for the `ryunix` language. See [VS Code](/en/docs/guides/vscode) for extension details.

## Generated structure

Typical output:

```txt
my-app/
├── app/
│   ├── index.ryx
│   ├── layout.ryx
│   ├── error.ryx      # global 404 (export NotFound)
│   └── api/hello/router.js
├── public/
├── styles/
├── ryunix.config.js
└── package.json
```

## After scaffolding

1. Run `pnpm dev` and open `http://localhost:3000`.
2. Adjust aliases in `ryunix.config.js` (`@/styles`, `@/components`, …).
3. Add routes under `app/` — folders map to URLs automatically.

## Canary vs stable

Documentation and examples often use `@unsetsoft/cra@canary` for the latest framework features. Pin a stable version for production apps when releases are tagged.

<CanaryChannelVersions />

With `--canary`, CRA runs `npm view @unsetsoft/ryunixjs@canary version` (and the same for presets) and writes semver ranges into `package.json`. With `--latest` (default), it resolves the `latest` dist-tag instead.
