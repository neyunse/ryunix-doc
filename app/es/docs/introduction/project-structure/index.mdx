# Estructura del proyecto

## Estructura básica

```
my-app/
├── app/
│   ├── layout.ryx         # Root layout (replaces main.ryx and App.ryx)
│   ├── index.ryx          # Root component (/)
│   ├── errors.ryx         # Error boundary
│   ├── sitemap.js         # Programmatic sitemap
│   ├── robots.js          # Programmatic robots.txt
│   ├── manifest.js        # Programmatic manifest
│   ├── api/               # API routes folder
│   │   └── route.js       # Back-end API endpoint
│   └── styles/            # CSS files
│       └── index.css
├── public/                # Static assets
│   ├── favicon.png
│   └── images/
├── ryunix.config.js       # Configuration
└── package.json
```

## Layout raíz (layout.ryx)

El archivo `layout.ryx` en la raíz de `app` envuelve todas tus páginas y ofrece una UI persistente al cambiar de ruta.

```jsx
import "@/styles/index.css";

function Layout({ children }) {
  return (
    <>
      <header>My Website</header>
      <main>{children}</main>
      <footer>© 2026</footer>
    </>
  );
}

export default Layout;
```

## Componente raíz (index.ryx)

```jsx
function Home() {
  return (
    <>
      <h1>Hello World</h1>
    </>
  );
}

export default Home;
```

## Organización recomendada de carpetas

### Arquitectura App Router

```
app/
├── layout.ryx
├── index.ryx
├── errors.ryx
├── sitemap.js
├── robots.js
├── manifest.js
├── about/
│   ├── layout.ryx
│   └── index.ryx
├── api/
│   └── users/
│       └── [id]/
│           └── route.js
├── components/       # Shared UI components
├── hooks/            # Custom hooks
├── utils/            # Utilities
└── styles/           # CSS
```



## Directorio public

Los archivos estáticos van en `public/`:

```
public/
├── favicon.png
├── images/
│   └── logo.png
├── fonts/
└── data/
    └── config.json
```

Acceso en código:

```jsx
<img src="/images/logo.png" />
```

## Archivo de configuración

```javascript
// ryunix.config.js
const RyunixSettings = {
  webpack: {
    production: false,
    output: {
      buildDirectory: ".ryunix",
    },
    resolve: {
      alias: {
        "@/": "./app",
        "@/components": "./app/components",
      },
    },
  },
  static: {
    favicon: true,
  },
};

export default RyunixSettings;
```

## Alias de importación

Configúralos en `ryunix.config.js` dentro de `webpack.resolve.alias`.

Uso en código:

```jsx
import Header from "@/components/Header";
```

## Variables de entorno

Crea un archivo `.env`:

```env
RYUNIX_APP_API_URL=https://api.example.com
RYUNIX_APP_KEY=abc123
```

Acceso en código:

```jsx
const apiUrl = ryunix.env.RYUNIX_APP_API_URL;
const key = ryunix.env.RYUNIX_APP_KEY;
```

Desde `ryunix.config.js`:

```javascript
const RyunixSettings = {
  env: {
    RYUNIX_APP_KEY: "XXXXXXXXXX",
  },
};
```

```jsx
const key = ryunix.config.RYUNIX_APP_KEY;
```

Todas las variables de entorno **deben** empezar por `RYUNIX_APP_` para exponerse al cliente.

## Buenas prácticas

1. **Mantén los componentes pequeños** - Responsabilidad única
2. **Usa carpetas por funcionalidad** - Agrupa archivos relacionados
3. **Nombres consistentes** - PascalCase para componentes
4. **Organiza por feature** - No por tipo de archivo
