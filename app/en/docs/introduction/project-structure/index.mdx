## Root Layout (layout.ryx)

The `layout.ryx` file in the root of your `app` directory wraps all your pages, providing a persistent UI across route changes.

```jsx
import "@/styles/index.css";

function Layout({ children }) {
  return (
    <>
      <header>My Website</header>
      <main>{children}</main>
      <footer>В© 2026</footer>
    </>
  );
}

export default Layout;
```

## Root Component (index.ryx)

```jsx
function Home() {
  return (
    <>
      <h1>Hello World</h1>
    </>
  );
}

export default Home;
```

## Configuration File

```javascript
// ryunix.config.js
const RyunixSettings = {
  webpack: {
    production: false,
    output: {
      buildDirectory: ".ryunix",
    },
    resolve: {
      alias: {
        "@/": "./app",
        "@/components": "./app/components",
      },
    },
  },
  static: {
    favicon: true,
  },
};

export default RyunixSettings;
```

## Import Aliases

Configure in `ryunix.config.js` inside `webpack.resolve.alias`.

Use in code:

```jsx
import Header from "@/components/Header";
```

## Environment Variables

Create `.env` file:

```env
RYUNIX_APP_API_URL=https://api.example.com
RYUNIX_APP_KEY=abc123
```

Access in code:

```jsx
const apiUrl = ryunix.env.RYUNIX_APP_API_URL;
const key = ryunix.env.RYUNIX_APP_KEY;
```

From `ryunix.config.js`:

```javascript
const RyunixSettings = {
  env: {
    RYUNIX_APP_KEY: "XXXXXXXXXX",
  },
};
```

```jsx
const key = ryunix.config.RYUNIX_APP_KEY;
```

All environment variables **must** start with `RYUNIX_APP_` to be exposed to the client.

## Best Practices

1. **Keep components small** - Single responsibility
2. **Use feature folders** - Group related files
3. **Consistent naming** - PascalCase for components
4. **Organize by feature** - Not by file type
